Build a full-stack Carpool & Ride-Sharing mobile-first web application 
for Pakistan using Next.js 14 (App Router) + Tailwind CSS + shadcn/ui.

The app is called "MAS Ride" with brand color: Purple (#7C3AED).

---

## CORE USER ROLES
1. Passenger — searches and joins carpool rides
2. Driver (Carpool Driver) — publishes and manages their carpool routes
3. Admin — manages users, rides, disputes (separate dashboard)

---

## AUTHENTICATION FLOW (screens in order)
1. Sign In screen — phone number input (+92 prefix) + Login button + 
   "Don't have an account? Register" link
2. Registration screen — phone number input + Sign Up button
3. OTP Verification screen — 6-box OTP input, 60s countdown timer, 
   "Resend Code" link, "Change Number" option, Verify button
4. Profile Setup — 3-step wizard with progress indicator:
   Step 1 "Informations": First name, Last name, Gender (dropdown), 
   Email, Date of Birth
   Step 2 "Address": Select City (dropdown), Select State (dropdown), 
   Country (default Pakistan)
   Step 3 "Profile": Avatar upload (camera icon), Done button + Skip option

## DATA MODEL (use mock data + localStorage for prototype)

User: { id, name, phone, email, gender, dob, city, state, avatar, 
        rating, totalRides, totalPosts, wallet, role, isDriver }

Ride: { id, driverId, vehicleType, origin, destination, stops, 
        departureTime, days, availableSeats, pricePerSeat, 
        routeComments, hasReturn, returnTime, returnPrice, 
        returnDays, status, passengers[] }

Booking: { id, rideId, passengerId, seats, status, totalPrice, 
           createdAt }

Message: { id, conversationId, senderId, text, timestamp }

## TECH STACK
- Next.js 14 (App Router)
- Tailwind CSS
- shadcn/ui components
- React Hook Form + Zod for validation
- Zustand for state management
- Google Maps API (or Leaflet as free alternative)
- Lucide React for icons
- date-fns for date handling
- Mock data with JSON (no backend needed for prototype)
